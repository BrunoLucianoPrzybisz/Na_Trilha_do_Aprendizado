SEU OBJETIVO:

Realizar o controle de versões de um código ou documento. É possivel gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedores.

SEUS TIPOS:

  1. CENTRALIZADO:

    2. Possui um repositorio central em um servidor com controle de verões.
    Dependente do servidor EX: Sofware Subrsion

  2. DISTRIBUÍDO:

    3. Possui um repositório local com controle de versôes.Atualização no repositorio central.
    Independente do servidor EX: Sofware Mercurial, Git

    GitHub

    Plataforma de hospedagem de código-fonte e arquivos com controle de versão.

MAS O QUE É GIT?

COMANDOS GIT:

  1. Commit = Torna permanente um conjunto de alteraçôes em um repositório.
  2. Push = Alterações para o servidor no repositório remoto.
  3. Fetch = Verifica se seu local está em sincronia com o remoto.
  4. Pull = sincroniza o repositorio local com o remoto atualizando as versões.
